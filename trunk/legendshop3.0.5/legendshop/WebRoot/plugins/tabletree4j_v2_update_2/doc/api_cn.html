Class 说明
Core4j.toolbox.TableTree4j
ConfigerObject
string:id
string类型,可选参数,默认为随机字符串,表格树的id,也是生成dom table对象的id.
boolean:isSortColumnReversal
布尔类型,可选参数,默认为false,针对sortByColumnIndex方法,对某列使用排序的时候,是否初始默认为反向.
string:scriptRootPath
string类型,可选参数,默认为自动获取,tabletree js所在根目录,一般不是改了tabletree js的文件名或者多个js文件同名,不需要设置.
string:themeName
string类型,可选参数,默认为'default',tabletree使用的外观,该参数与'theme/'路径下的文件夹名一致,所以如果要增加theme外观,可在theme目录下新建自定义的外观文件夹,具体实现可以参看里面已有的外观文件夹.
string:cookiesSplitString
string类型,可选参数,默认为'#T#',tabletree所用到cookies保存展开节点的id值之间的分隔符,一般不需要设置.
string:treeMode
string类型,可选参数,默认为'menu',tabletree的模式,只有'menu'和'gird'两种,'menu'为单列无边框的菜单模式,'gird'为多列有边框的表格模式,实际上你可通过覆盖css属性的方法重新定义tabletree的显示样式.
string:renderTo
string类型,可选参数,tabletree创建到dom元素的id,默认创建到dom body对象下
[object]:columns
对象数组类型,必须参数,定义tabletree每一列的信息,每列的对象包括以下属性:
{isNodeClick:,dataIndex:,width:,renderFunction:,canSort:,isDefalutSort:}
isNodeClick:布尔类型,可选参数,是否为树结构所在列,至少有一列要设置为true;
dataIndex:string类型,必须参数,渲染列每一行数据对象的属性名;
width:string类型,可选参数,设置列的宽度;
renderFunction:function类型,可选参数,渲染列每一行的函数,其接收的参数为一个包含多个属性信息的对象.
{dataValue:,node:,tabletreeObj:,rowObj:,rowIndex:,container:,columnIndex:}
dataValue:数据的值;
node:对应的节点Core4j.toolbox.TableTreeNode对象;
tabletreeObj:当前的tabletree对象;
rowObj:dom的TR对象;
rowIndex:int型,当前行的索引;
container:dom的元素对象,renderFunction渲染结果最后包含在该元素下;
columnIndex:int型,当前列索引.
canSort:布尔类型,可选参数,但前列是否可以被排序;
isDefalutSort:布尔类型,可选参数,如果但前列可以被排序,是否是默认的排序列.
[Core4j.toolbox.TableTreeHF]:headers
Core4j.toolbox.TableTreeHF对象数组类型,可选参数,定义多个header,每个header相当于一行,TableTreeHF对象包括以下属性:
{columns:[{dataIndex:,tdAttributeNames:,tdAttributeValueObject:}],dataObject,trAttributeNames:,trAttributeValueObject:}
columns:对象数组类型,必须参数,定义header的每一列信息,每列的对象包括以下属性:
{dataIndex:,tdAttributeNames:,tdAttributeValueObject:}
dataIndex:string类型,必须参数,渲染列所在行的单元格数据对象的属性名;
tdAttributeNames:字符串数组类型,可选参数,为列所在行的单元格添加属性名(多个);
tdAttributeValueObject:字符串数组类型,可选参数,,为列所在行的单元格添加属性名对应的值(多个).
dataObject:header绑定的数据对象;
trAttributeNames:字符串数组类型,可选参数,为header所在行添加属性名(多个);
trAttributeValueObject:字符串数组类型,可选参数,,header所在行添加属性名对应的值(多个).
[Core4j.toolbox.TableTreeHF]:footers
Core4j.toolbox.TableTreeHF对象数组类型,可选参数,定义多个footer,每个footer相当于一行,定义方式如同上面的header一样.
int:cookieTime
int型,可选参数,默认值为30*24*60*60*1000毫秒(30日),tabletree中使用cookies的生存时间,单位为毫秒.
boolean:floatRight
布尔类型,可选参数,默认值为false,tabletree中所有列的内容从右到左显示.
boolean:useCookie
布尔类型,可选参数,默认值为false,tabletree是否使用cooike保存已经展开的节点.
boolean:useIcon
布尔类型,可选参数,默认值为true,tabletree是否显示节点图标.
boolean:useLine
布尔类型,可选参数,默认值为true,tabletree是否显示节点的树状结构线.
boolean:useOrder
布尔类型,可选参数,默认值为true,tabletree是否根据节点设定的order值排序.
string:selectMode
string类型,可选参数,默认值为'muti',tabletree的行选择模式,有3种,分别为'muti'多选(按下ctrl进行多选),'single'单选,'none'不能选择行.
[object]:plugins
对象数组类型,可选参数,对tabletree进行扩展的plugin对象,这些对象必须实现一个initTableTree4jPlugin(tabletreeObj)方法,tabletree初始化的时候会先加载这些plugin,并调用它们的 initTableTree4jPlugin方法对plugin进行初始化.

string类型,必须参数,节点的id,也是生成dom table的tr对象的id.

Css 说明
必须引入根目录下的tabletree4j.css,如果要修改tabletree的边框颜色和粗细,间隔等,可以通过覆盖tabletree4j.css中的style实现;
另外js还会根据你选择的theme自动引入根目录下theme/[你设置的theme]/css/theme.css 文件,里面定义了行高,图标的宽度高度等等, 你可以通过覆盖你当前theme的style实现你想要的样式,也可以参考theme下的目录,实现自己的theme.


目录结构和文件引入
目录结构
demo/: 存放所有的demo
doc/: 存放组件的文档说明
src/: 存放组件未经压缩的原始js代码
theme/: 存放组件的外观样式包
plugin/: 存放组件的plugin js文件
* 根目录下的TableTree4j.js和Core4j.js是经过压缩的js代码
* 根目录下的TableTree4j-debug.js和Core4j-debug.js是未经压缩的原始js代码,方便调试,测试的时候可引入这两个,正式应该使用经过压缩的js

文件引入
依次分别引入根目录下的
Core4j.js
tabletree4j.css
